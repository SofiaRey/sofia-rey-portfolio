# Implementation Plan — Sofía Rey Portfolio

## Overview

Single-page portfolio website for Sofía Rey (multimedia designer) with an Apple-style scroll-driven video frame animation as the full-page background. Built with Bun + React + Tailwind CSS.

## Architecture

### Three-layer visual stack

1. **Canvas (z-0)** — Fixed full-viewport `<canvas>` rendering pre-extracted JPEG frames
2. **Content (z-10)** — Page sections with transparent backgrounds, floating over the canvas
3. **Noise overlay (z-9999)** — Animated film-grain texture, `pointer-events: none`

### Frame Sequence System

- **Source:** `src/assets/video.mp4` (2560x1440, 27s, 30fps)
- **Extraction:** `bun run extract-frames` → 378 JPEG frames at two resolutions
  - Desktop: 1920x1080 → `public/frames/desktop/0001.jpg`–`0378.jpg`
  - Mobile: 960x540 → `public/frames/mobile/0001.jpg`–`0378.jpg`
- **Breakpoint:** ≤768px uses mobile set

### Playback Modes

1. **Intro Playback (frames 0–149):** Auto-plays over 5 seconds on page load. Starts after first 30 frames preload; remaining intro frames load during playback. Scroll is ignored during this phase.
2. **Scroll-Driven Mode (frames 150–377):** After intro completes, frame index = `150 + (scrollY / maxScroll) * 228`. If user scrolled during intro, canvas jumps to correct frame on transition.

### Frame Loading Strategy

- **Phase 1:** Preload first 30 intro frames eagerly (parallel Image() requests). Intro starts once these are loaded.
- **Phase 2:** Remaining intro frames (30–149) load concurrently during the 5s playback. drawFrame uses nearest available frame for any not yet loaded.
- **Phase 3:** Background-load 228 scroll frames (150–377) after all intro frames are done. If a scroll frame isn't loaded yet, show nearest available frame.

## Page Sections

| # | Section | ID | Notes |
|---|---------|-----|-------|
| 1 | Nav | — | Fixed, transparent→blur on scroll, active section tracking |
| 2 | Hero | `#inicio` | "soy ___?" with vertical-slide role rotation (3s interval), aligned to bottom of viewport |

| 5 | Procesos Compartidos | — | 2x2 grid (desktop), auto-scroll carousel (mobile), 4 photos |
| 6 | Contact | `#contacto` | Form UI → `mailto:` link on submit |
| 7 | Footer | — | LinkedIn + GitHub links |

## Responsive Strategy

- **Desktop (>768px):** Full layout, 1920x1080 frames
- **Mobile (≤768px):** Single-column stack, 960x540 frames, hamburger nav → full-screen overlay

## Typography & Color

- **Serif (headings):** Libre Caslon Text (italic) — Google Fonts
- **Sans-serif (body):** Kedebideri — Google Fonts
- **Black:** `#000000`
- **Red accent:** `#C12D2D`
- **Background:** `#FFFFFF`

## Animations

- **Role rotation:** Vertical slide, 3s per word, infinite loop
- **Noise overlay:** Animated canvas-generated grain, ~4% opacity, 100ms refresh
- **Process items:** Orbit in a circle (20s CSS rotation), counter-rotate to stay upright, scale to 1.25 on hover, circle pauses on hover
- **Content reveals:** Fade-up (opacity 0→1, translateY 20px→0) via Intersection Observer, triggered once at 20% viewport entry
- **Photo carousel (mobile):** CSS marquee animation, continuous loop, ~35s cycle

## Build & Deploy

- **Dev:** `bun --hot src/index.ts` (port 3002, HMR)
- **Build:** `bun run build` (static output to `dist/`)
- **Frames:** `bun run extract-frames` (requires ffmpeg, output gitignored)
- **Deploy:** Static hosting (Vercel/Cloudflare Pages). Build command: `bun run extract-frames && bun run build`
